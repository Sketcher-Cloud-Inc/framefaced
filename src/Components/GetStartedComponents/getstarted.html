Create environment variables

You need to configure your environment variables before launch your project. You can use file .env.sample and rename it .env to create the default environment. Environment variables file is based on Symfony\Dotenv, you can use the official documentation for more informations about syntax.

Debug ModeVariable name: DEBUG_MODE

Important, if you're in debug mode you need to create a second file .env.dev. You can overwrite defined variables in .env, or leave it blank.

Example

# Enable debug mode (display errors and load ".env.dev")
DEBUG_MODE=true

DatabasesVariable name: DATABASE_{{dbname}}_USER & DATABASE_{{dbname}}_PASSWORD

For each databases referenced in your databases.json you need to specify username and password. Replace content {{dbname}} by your database name in databases.json. You can leave password as blank, but you need to declare it.

Example

# Example of database credentials
DATABASE_EXAMPLE_USER=example
DATABASE_EXAMPLE_PASSWORD=example-password

Authentication token database locationVariable name: AUTH_TOKEN_DBNAME & AUTH_TOKEN_TABLE_NAME

In this variable you need to specify the authentication token database and table name.

Example

# Authentication token database location
AUTH_TOKEN_DBNAME=example
AUTH_TOKEN_TABLE_NAME=tokens

Run a development server

You can use the command run in your project console (like this: ./bin/console run).

Hostname & Port

You can specify the hostname and the port with ./bin/console run {{hostname}}:{{port}}
